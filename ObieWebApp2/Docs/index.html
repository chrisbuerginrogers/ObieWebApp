Getting Started — Data Folder

All three tools are browser-based and run entirely locally — nothing is uploaded to any server. They use the File System Access API (Chrome or Edge 86+ required) to read and write files in a folder you choose on your computer.

Select a Data Folder

Before using any tool, click the Data Folder button (on the home page or inside any tool) and select the folder that holds your measurement files. The folder name is remembered between visits. Grant read/write access when the browser asks — this allows the tools to load FRF files and save new measurements automatically.

Folder layout expected by the tools Measurement files (AvR, AvC, TRF, CSV) can be anywhere inside the Data Folder. Acquire saves each run into a timestamped sub-folder. Settings files (templates, band presets, lists) are stored in ObieAppSettings/ inside your Data Folder and are loaded automatically on startup.

Supported file formats

ExtensionDescription
.avr / .avcAveraged FRF (real/imaginary columns) — main format for Explore and Convolve It
.trf / .trvTransfer-function format — also supported by Explore and Convolve It
.csvTwo-column CSV (frequency, magnitude dB) — quick import for Convolve It
.wavAudio file used as the dry signal in Convolve It; also storable as a sound snippet in Explore

Explore

PyScript
Open Explore →

Explore lets you load multiple FRF files at once, overlay them on a single interactive plot, and compare them with smoothing, normalization, band averaging, and per-dataset convolution playback.

Loading files

Three ways to load FRF files:

  1. Browse — opens a file picker; select one or more FRF files directly.
  2. Search — opens the Search modal. Type a filename pattern (comma-separated for OR logic), filter by file type, and load all matching files from your Data Folder with one click.
  3. Drag & drop — drag files from your desktop onto the plot area.

Band averaging

Select a band preset from the Bands dropdown (presets are loaded from ObieAppSettings/bands/ in your Data Folder). The band table below the dataset list shows the average dB level for each dataset in each frequency band. Toggle shade to highlight the band regions on the plot. Click Excel to export the band data as a CSV file.

Interpret

Click Interpret to open a reference overlay showing the standard Oberlin mode labels (A0, CBR, B1−, B1+, Bridge Hill, etc.) with their typical frequency ranges drawn on a copy of the current plot. Includes links to violinacoustics.com and euphonics.org.

Convolve It

PyScript
Open Convolve It →

Convolve It applies a measured FRF to a dry audio file using full complex frequency-domain convolution: Y(f) = H(f) · X(f). The result sounds like the dry signal played through (or by) the measured instrument.

Step-by-step workflow

  1. Load an FRF. Under FRF Files, click Left / Mono FRF and pick an AvR, AvC, TRF, or CSV file. For stereo output, also load a Right FRF; the tool then produces a two-channel result with independent left and right filtering.
  2. Load a WAV. Under Signal (WAV), click Load WAV and select any WAV file (mono or stereo). Use ▶ Play WAV to confirm the dry signal.
  3. Set the output gain. Drag the Output gain slider (−24 to +24 dB) to avoid clipping or to match levels.
  4. Convolve. Click ⚡ Convolve. The button is enabled only once both an FRF and a WAV are loaded.
  5. Listen and save. Press ▶ Play Result to audition, then 💾 Save to download the convolved WAV to your computer.

The four plots

FRF · Magnitude (dB)
  • Shows the loaded FRF on a log-frequency axis.
  • Both L and R channels are drawn if stereo.
Input · Spectrogram
  • Time-frequency spectrogram of the dry WAV.
  • Toggle dB / linear amplitude with Spec: dB in the toolbar.
Output · Convolved Waveform
  • Time-domain waveform of the convolved result.
  • Useful for checking for clipping.
Output · Spectrogram
  • Spectrogram of the convolved result.
  • Compare with the input spectrogram to see the FRF shaping effect.
Technical details Convolution is performed in the frequency domain using a Hann-windowed FFT. The FRF is interpolated onto the audio-sample frequency grid using log-space interpolation. The result is reconstructed with minimum-phase or zero-phase methods as appropriate for the file type.

Acquire

PyScript
Open Acquire →

Acquire turns your laptop and a USB audio interface into an impact-hammer FRF measurement system. It records hammer and microphone signals simultaneously, detects hits automatically, averages multiple hits per position, and saves each position as an AvR CSV file.

Hardware setup

  1. Connect your USB audio interface (e.g. Focusrite Scarlett 2i2) before opening the page.
  2. Plug the impact hammer into Input 1 and the microphone into Input 2 (or swap in Settings if your hardware is wired differently).
  3. Use Chrome or Edge — Safari and Firefox do not support the required audio APIs.

First-time setup (Settings)

Click Settings in the toolbar and configure:

SettingNotes
Input deviceChoose your audio interface from the dropdown. Defaults to the system default if left blank.
Soundcard typeFree-text label saved into each output file for traceability.
Trigger thresholdVoltage level (V) that triggers a hit. Lower values catch lighter taps; raise it to ignore ambient noise.
Time before / after hitPre- and post-trigger capture window in seconds.
Hammer / Mic time cutoffSignal is zeroed after this time — suppresses late reflections. Shown as a green line on the mini plots.
Hammer calibration (N/V)Converts raw voltage to Newtons. Enter the value from your hammer's calibration sheet.
Mic calibration (Pa/V)Converts raw voltage to Pascals. Enter the value from your microphone calibration sheet.
Hits per positionNumber of good hits to average before advancing to the next position.
Number of positionsTotal positions in the measurement run (e.g. 12 for a standard plate grid).
Position prefixShort label prepended to position names (e.g. "H" gives H1, H2, …).

Click Apply & Save. Settings are stored locally in the browser and also saved to your Data Folder template.

Running a measurement

  1. Click 📁 Data Folder and select the folder where results should be saved. A new timestamped run sub-folder is created automatically on first hit.
  2. Enter the Instrument and Test name in the banner fields.
  3. Click ▶ Start. The app begins listening for hammer hits.
  4. Strike the instrument. The three mini plots (Hammer FFT, Hammer time, Mic time) update after each hit. Check that the hammer pulse is clean — a double-hit shows two bumps in the Hammer time plot.
  5. After the required number of good hits the app auto-advances to the next position and saves the averaged FRF to your Data Folder.
  6. Use ↩ Undo Hit to remove a bad last hit, or 🗑 Clear to restart the current position. ↺ Start over clears all positions.
  7. When all positions are complete, click ■ Stop.

Reviewing the FRF plot

The main plot on the right shows all completed positions overlaid in different colors. Use the plot-toolbar controls to toggle log/linear X axis, rescale Y, or adjust the dB range. Hover over any curve to read out the exact frequency and level.

Templates

A template is a JSON file that stores your complete measurement setup (soundcard, threshold, calibration factors, hit count, position count, etc.). Templates are loaded automatically from ObieAppSettings/Templates/ inside your Data Folder when Acquire starts.

  1. Click Template… to open the template manager.
  2. Select a template from the list (it will preview the settings as JSON), then click Apply.
  3. To save the current settings as a new template, click Save as Template.
  4. Use Browse… to import a template file from outside your Data Folder.

Session Notes

Click Notes to open the Notes panel. Enter any free-text observations about the measurement session. Notes are saved per run-folder name in the browser and persist across reloads.

LiveView

Click LiveView to open a large full-screen view of the FRF plot — useful when the laptop is placed away from the instrument during measurement.

Tip — adjusting the threshold on the fly You can change the trigger threshold directly in the Hammer mini-plot header without opening Settings. Edit the number next to V and press Enter to apply immediately. The purple dotted threshold line on the Hammer time plot updates in real time.
